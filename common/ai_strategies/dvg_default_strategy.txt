	# A fleet composition describes a role a fleet can take (e.g. main_battle_fleet, raid_fleet):
	# what kind of ships it should be made of and what missions it should pursue. The AI will try
	# to match each of the country's existing fleets to the role they best fit.
	# All numeric fields accept script values, so weights can be tuned from country scope.
	#
	# ship_group_weights:
	#   Per ship group desire for this role. Across all available roles, weights are summed per
	#   group, and each role's share becomes its target ratio of the country's total ships in
	#   that group. Existing fleets are scored against every role's target ratios and assigned to
	#   the role they fit best, so these weights drive both "how should the country's ships be
	#   split across roles" and "which fleet plays which role".
	#
	# mission_type_weights:
	#   Multiplier applied when a fleet in this role picks which mission to pursue. 1.0 is
	#   neutral, higher biases the fleet toward that mission, 0.0 disables it for this role.
	#   IMPORTANT: if the block is present, any mission type not listed is treated as 0, so the
	#   list gates which missions this role will take. Omit the whole block to leave every
	#   mission neutral.
	#
	# min_ships_wanted:
	#   Soft floor on fleet size used during role-fit scoring. Fleets smaller than this take a
	#   penalty when matched to this role; it does not disqualify them and is not a hard cap.
	#
	# potential:
	#   Country-scoped trigger. If it fails the role is dropped entirely, so no fleet will be
	#   assigned to it.
	#
	# replace_key:
	#   Optional key of another composition that this one should replace when both are collected
	#   from the country's active strategies. Use it in a country-specific strategy to override a
	#   default role (e.g. a custom raid_fleet with replace_key = raid_fleet swaps out the one
	#   defined here). Without a replace_key the composition is added alongside the existing set.
